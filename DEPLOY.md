# 后台管理页面部署到宝塔Linux面板步骤

## 一、服务器环境准备

### 1. 登录宝塔Linux面板
1. 打开浏览器，访问宝塔面板地址：`http://124.222.114.63:8888`（默认端口8888，如已修改请使用实际端口）
2. 输入用户名和密码登录宝塔面板

### 2. 安装Nginx
1. 登录后，进入「软件商店」
2. 在搜索框中输入「Nginx」
3. 找到Nginx后，点击「安装」
4. 选择最新稳定版本，点击「提交」
5. 等待安装完成

## 二、创建网站

### 1. 添加站点
1. 进入「网站」菜单
2. 点击「添加站点」
3. 填写站点信息：
   - 域名：如果有域名，填写域名；如果没有，填写服务器IP `124.222.114.63`
   - 备注：填写「免费车登记后台管理系统」
   - 根目录：默认即可，或自定义（如 `/www/wwwroot/admin.example.com`）
   - PHP版本：选择「纯静态」（因为是纯HTML/CSS/JS应用）
   - 数据库：无需创建
   - FTP：根据需要选择是否创建，建议创建以便后续文件上传
   - SSL：如果有域名，建议申请SSL证书启用HTTPS
4. 点击「提交」创建站点

## 三、上传项目文件

### 方式一：使用宝塔面板文件管理器上传
1. 进入「网站」菜单，找到刚创建的站点，点击「根目录」进入文件管理器
2. 删除根目录下的默认文件（如 `index.html`、`404.html` 等）
3. 点击「上传」按钮，选择本地项目中的 `admin` 目录下的所有文件（或 `dist-admin` 目录下的所有文件）
4. 等待上传完成

### 方式二：使用FTP工具上传
1. 如果创建站点时启用了FTP，使用FTP工具（如FileZilla）连接服务器
2. 连接信息：
   - 主机：`124.222.114.63`
   - 端口：21（默认）
   - 用户名：创建站点时设置的FTP用户名
   - 密码：创建站点时设置的FTP密码
3. 连接成功后，将本地项目中的 `admin` 目录下的所有文件（或 `dist-admin` 目录下的所有文件）上传到服务器站点的根目录

### 方式三：使用命令行上传（适用于Linux/Mac本地环境）
```bash
# 使用scp命令上传文件
scp -r /path/to/your/project/admin/* root@124.222.114.63:/www/wwwroot/your-site-domain/
```

## 四、配置Nginx（可选）

### 1. 修改Nginx配置
1. 进入「网站」菜单，找到刚创建的站点，点击「设置」
2. 进入「配置文件」选项卡
3. 根据需要修改Nginx配置，例如：
   - 添加gzip压缩
   - 设置缓存策略
   - 配置HTTPS（如果有SSL证书）
4. 点击「保存」

### 2. 重启Nginx
1. 进入「软件商店」
2. 找到已安装的Nginx，点击「设置」
3. 点击「重启」

## 五、测试访问

### 1. 访问后台管理系统
1. 打开浏览器，访问站点地址：
   - 如果使用IP访问：`http://124.222.114.63`
   - 如果使用域名访问：`http://your-domain.com`（或 `https://your-domain.com` 如果启用了HTTPS）
2. 应该能看到后台管理系统的登录页面

### 2. 登录测试
1. 输入用户名和密码进行登录（初始用户名密码请参考项目文档或数据库配置）
2. 登录成功后，应该能看到后台管理系统的主页面

## 六、后续维护

### 1. 更新代码
- 当本地代码更新后，重复「上传项目文件」步骤，将新的代码上传到服务器
- 如果修改了Nginx配置，需要重启Nginx

### 2. 备份数据
- 定期备份站点文件和数据库（如果有）
- 宝塔面板提供了「备份」功能，可以设置自动备份

### 3. 监控站点
- 使用宝塔面板的「监控」功能，查看站点的访问情况和服务器资源使用情况
- 设置告警，及时发现并处理问题

## 七、常见问题处理

### 1. 页面无法访问
- 检查服务器防火墙是否开放了80端口（HTTP）和443端口（HTTPS）
- 检查Nginx是否正在运行
- 检查站点配置是否正确

### 2. 页面样式错乱或功能异常
- 检查文件上传是否完整
- 检查浏览器控制台是否有错误信息
- 检查网络连接是否正常

### 3. 登录失败
- 检查用户名和密码是否正确
- 检查Supabase配置是否正确（查看 `admin/config.js` 文件中的Supabase配置）
- 检查Supabase服务是否正常

## 八、项目说明

### 1. 项目结构
- `admin/`：后台管理系统源代码
- `dist-admin/`：构建输出目录（如果有）
- 后台管理系统是纯静态HTML/CSS/JavaScript应用，使用Supabase作为后端服务

### 2. 技术栈
- HTML5 + CSS3 + JavaScript
- Supabase（后端服务）
- 纯静态应用，无需服务器端渲染

### 3. 主要功能
- 登录验证
- 登记记录管理
- 分公司管理
- 收费站管理
- 班组管理
- 收费员管理
- 监控员管理
- 班次设置
- 用户管理

## 九、注意事项

1. 确保服务器安全组开放了80/443端口
2. 定期更新服务器和软件
3. 定期备份站点文件和数据库
4. 设置强密码，保护宝塔面板和FTP账号
5. 如启用HTTPS，定期更新SSL证书
6. 监控站点访问情况，及时发现异常访问

以上就是将后台管理页面部署到宝塔Linux面板的完整步骤。按照上述步骤操作，即可成功部署后台管理系统。
